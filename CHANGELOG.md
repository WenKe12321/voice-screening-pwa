# 更新日志

## v0.4.1 - 2026-06-21

用户友好筛查与开源项目调研版。

### 新增

- 新增 `docs/user-friendly-risk-and-open-source-review.md`，整理抑郁情绪辅助判断机制、用户友好设计原则和 GitHub 开源项目调研。
- 明确 PHQ-9 仍是唯一风险等级来源，第 9 题非零时应优先展示求助资源。
- 提出功能受损程度、历史趋势、录音质量反馈和本地个人支持计划等后续改进方向。
- 筛选 `wavesurfer.js`、安全计划应用、情绪日记和 PHQ-9 原型等开源项目，并标注可借鉴程度和风险。

## v0.4.0 - 2026-06-08

多层嵌套学习框架版。

### 新增

- 新增五层嵌套学习框架：语音片段特征、任务级表示、个体筛查模型、中文大学生目标域适配、持续评估更新。
- `.vmodel` 支持可选 `modelCard.nestedLearning` 元数据，前端导入时校验框架版本、目标人群、层级顺序和校准状态。
- 设置页新增“多层嵌套学习框架”说明，导入模型后展示当前模型阶段。
- 训练流水线导出的报告和候选 `.vmodel` 会写入同一套嵌套学习元数据。
- 新增 `docs/nested-learning-framework.md`，明确当前 EATD 只属于公开中文基线，尚未完成中文大学生 PHQ-9 目标域校准。

## v0.2.1 - 2026-06-02

录音稳定性修复版。

### 修复

- 录音改为生成完整音频容器，避免 Edge 偶发提示 `Unable to decode audio data`。
- 优先选择 `audio/webm;codecs=opus`，并保留 WebM 与 MP4 兼容回退。
- 录音读秒改为基于真实开始时间计算，避免浏览器调度延迟导致计时漂移。
- 录音计时显示调整为 `MM:SS` 格式。
- 本地解码分析期间锁定操作按钮，避免重复点击造成状态交错。
- 解码失败时展示可操作的中文重录提示。

### 验证

- 新增录音器完整容器、真实时钟读秒和停止后计时清理测试。
- 手机端 Chromium E2E 增加 `00:01` 读秒断言，并继续验证本地解码、离线重开、加密导出和零外部请求。

## v0.2.0 - 2026-06-01

研究模型分阶段训练版。

### 新增

- 增加 EATD-Corpus 中文开放回答真实数据训练流水线。
- 增加 Androids Corpus 朗读与访谈任务分离的五折离线评估。
- 增加逻辑回归、SVM、随机森林和 GRU/BiLSTM 离线研究对照。
- 增加研究采集模式，使用积极、中性、消极三个开放回答任务。
- 增加 `.vmodel` 校验、IndexedDB 存储和端侧线性推理适配器。
- 增加 MODMA EULA 申请说明、数据校验测试和去标识化实验报告。

### 隐私与边界

- 原始语音、派生特征、模型权重和逐样本预测只保存在本地 `F:\Datasets\voice-screening`。
- 仓库不包含 EATD、Androids、MODMA 等数据集或派生权重。
- PHQ-9 仍是应用内唯一风险等级来源。
- 语音模型仅用于非商业学术研究，不提供医学诊断。
- EATD 候选模型未达到启用门槛，因此当前版本不附带真实可导入 `.vmodel`。
