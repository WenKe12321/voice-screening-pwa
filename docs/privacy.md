# 隐私设计与风险边界

## 默认行为

应用不需要账号，不接入远程分析服务，不自动上传录音。所有敏感内容均保存在当前浏览器的 IndexedDB 中。

敏感内容包括：

- PHQ-9 回答和总分。
- 语音特征摘要和未验证演示指数。
- 四段原始录音。
- 用户主动填写的匿名研究编号。

## 本地加密

首次使用时，用户创建至少 8 个字符的访问口令。应用使用：

- `PBKDF2-SHA-256`，迭代次数 `240000`，从口令派生密钥。
- `AES-GCM-256`，使用随机 salt 与 IV 加密会话。
- 不持久化派生密钥，不上传访问口令。

关闭应用后需要重新输入口令。口令遗失无法恢复，只能清空本地保险箱。

## 删除与导出

- 用户可以删除单条筛查记录。
- 用户可以一键清空整个本地保险箱。
- 用户主动导出的 `.vscreen` 研究包仍为加密文件。
- 新版 `.vscreen` 研究包包含独立解密所需的随机 salt、迭代次数和加密校验器，不包含访问口令或派生密钥。
- 用户可以在本项目目录中运行 `npm run decrypt:vscreen -- "<文件路径>"`，输入访问口令后在本地解密研究包。
- 应用不会自动导出或上传研究包。

## 高风险提示

本应用不能处理紧急情况，也不能替代专业服务。PHQ-9 第 9 题只要非零，无论总分多少，应用都会提示：

- 联系身边可信任的人。
- 拨打全国心理援助热线 `12356`。
- 存在即时危险时联系 `120` 或 `110`。

## 已知限制

- 浏览器存储可能被用户清理，也可能受设备策略影响。
- 本地加密降低了浏览器存储被直接读取的风险，但不能替代设备锁屏、系统更新和良好的口令习惯。
- 浏览器麦克风只能在 HTTPS 或 `localhost` 环境下使用。
