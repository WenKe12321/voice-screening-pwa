# 抑郁情绪辅助判断、用户友好设计与开源项目调研

调研日期：2026-06-21

## 结论摘要

本项目应继续采用“辅助筛查”而不是“诊断判断”的产品边界。对用户是否可能存在抑郁情绪的判断，应由 PHQ-9 风险分层承担；语音特征、历史趋势和开放回答只能作为研究性辅助信息，不能改变 PHQ-9 等级。

对有抑郁情绪的用户更友好的方向不是增加刺激性提醒或强诊断结论，而是降低认知负担、减少羞耻感、给出可执行支持、保持隐私透明，并在安全信号出现时提供最短路径求助。

## 抑郁情绪辅助判断机制

### 1. 主判断：PHQ-9 总分

PHQ-9 的 9 个题目每题 0-3 分，总分 0-27。应用内风险等级应继续只来自 PHQ-9：

| PHQ-9 总分 | 应用展示 | 建议语气 |
|---|---|---|
| 0-4 | 当前困扰较少 | 鼓励继续关注睡眠、情绪和节奏 |
| 5-9 | 存在轻度困扰 | 建议休息、记录趋势、与可信任的人聊聊 |
| 10-14 | 存在中度困扰 | 建议近期联系学校心理中心或专业人员 |
| 15-19 | 困扰程度较高 | 建议尽快联系专业支持 |
| 20-27 | 困扰程度较高 | 建议尽快联系专业支持，并显示更明显的求助资源 |

页面文案不应写“你有抑郁症”，而应写“你的回答提示近期可能存在一定程度的情绪困扰”。PHQ-9 可以用于筛查和严重程度评估，但诊断仍需要专业人员结合访谈和背景信息判断。

### 2. 安全兜底：PHQ-9 第 9 题

第 9 题只要非 0，就应优先展示求助卡片，不等待总分达到高分：

```ts
if (phq9Item9 > 0) {
  showUrgentSupportCard = true
}
```

求助卡片应包含：

- 全国心理援助热线 `12356`。
- 已配置的校内心理中心联系方式。
- 即时危险时联系 `110`、`120` 或身边可信任的人。
- 简短提示：“如果你正处于即时危险中，请优先寻求现实帮助，不要继续独自使用应用。”

### 3. 解释增强：功能受损程度

建议后续增加 PHQ-9 常见附加问题：

> 这些状态对你的学习、生活或人际关系造成了多大影响？

选项可以是：

- 没有影响。
- 有一点影响。
- 有明显影响。
- 影响很大。

这个问题不计入 PHQ-9 总分，但可以用于调整建议语气。例如，轻度分数但功能影响很大时，建议主动联系校内心理中心；中度分数但功能影响较小，也应建议持续观察和复测。

### 4. 研究辅助：语音特征

语音模块只能展示研究性信号：

- 有效语音比例。
- 停顿比例。
- RMS 能量均值与波动。
- 过零率。
- 基频中位数与范围。
- 频谱质心。
- MFCC 摘要。
- 语速代理指标。

语音结果不得写成“语音判断你抑郁”。更安全的表达是：

> 本次录音显示若干可研究的声学模式。这些特征不能单独判断抑郁状态，也不改变 PHQ-9 风险等级。

### 5. 趋势判断：多次自测变化

单次结果容易受考试、睡眠、身体状态和录音环境影响。后续可以增加趋势提示：

| 趋势 | 展示方式 |
|---|---|
| 连续上升 | “近期困扰可能有加重趋势，建议主动寻求支持。” |
| 连续下降 | “近期困扰可能有所缓解，仍建议继续关注状态。” |
| 波动明显 | “状态存在波动，可以结合近期生活事件理解。” |
| 任意一次第 9 题非 0 | 始终显示求助卡片。 |

趋势判断只用于温和提示，不应作为诊断依据。

## 用户友好设计原则

### 低认知负担

- 一屏只做一个主要决定。
- 避免大段解释堆在录音和结果页。
- 对“下一步做什么”给出明确按钮。
- 在正式自测、快速体验、答辩展示之间保持清楚区分。

### 降低羞耻感

- 使用“困扰”“状态”“支持”，避免“异常”“病人”“危险人格”等标签。
- 结果页先承认“这不是诊断”，再给建议。
- 不把结果做成红色警报式评分榜。

### 危机友好

- 第 9 题非 0 时，求助入口必须比解释文字更显眼。
- 求助卡片使用电话号码链接，减少复制和搜索负担。
- 不使用聊天机器人替代危机支持。
- 不让用户为了获取帮助而必须先完成更多表单。

### 隐私透明

- 明确说明录音、问卷、模型结果不自动上传。
- 不嵌入广告、追踪 SDK 或第三方分析。
- 导出研究包必须由用户主动触发。
- 对 `.vmodel` 说明来源、适用任务和验证指标。

### 自主感

- 提供正式自测、快速体验、删除、导出、清空数据。
- 允许用户不填写匿名研究编号。
- 允许用户只查看帮助资源，不强迫完成筛查。

## 论文与资料线索

| 来源 | 与本项目相关的要点 | 项目启示 |
|---|---|---|
| Kroenke 等，PHQ-9 validity of a brief depression severity measure | PHQ-9 是常用抑郁症状筛查和严重程度量表 | 继续以 PHQ-9 作为唯一风险等级来源 |
| PHQ-9 说明与计分资料 | 常见分层为 0-4、5-9、10-14、15-19、20-27 | 结果页继续使用分层而非诊断标签 |
| Haque & Rubya, mental health app reviews | 用户批评心理健康应用缺少透明度、危机支持不足、聊天机器人不可靠 | 应用不做危机聊天机器人，优先提供明确求助资源 |
| Slovak 等，emotion regulation HCI | 情绪调节技术需要理论依据、机制设计和实际实现之间保持一致 | 任何安抚提示都应轻量、可解释，不做未经验证的干预 |
| Iwaya 等，mental health app privacy | 心理健康应用存在权限、加密、日志和第三方共享风险 | 坚持端侧处理、无追踪、无自动上传 |
| Saini 等，mobile mental health apps privacy/security | 移动心理健康应用可能存在危险权限、弱安全和流量泄露 | 继续保持 PWA 本地处理路线，避免不必要权限 |

## 对当前项目的改进建议

### 优先级 P0

1. 保持 PHQ-9 为唯一风险等级来源。
2. 第 9 题非 0 时显示更短、更显眼的求助路径。
3. 继续禁止自动上传录音和问卷。
4. 所有语音模型输出都标注为研究概率或演示指数。

### 优先级 P1

1. 新增“功能受损程度”附加问题。
2. 新增历史趋势卡片，显示最近 3 次 PHQ-9 变化。
3. 新增“我现在需要帮助”常驻入口，不要求先完成自测。
4. 引入录音质量反馈，例如音量过低、环境过吵、录音太短。

### 优先级 P2

1. 评估 `wavesurfer.js` 用于历史录音波形预览。
2. 参考安全计划应用，设计本地“个人支持计划”页面。
3. 添加可选情绪日记，但不把日记文本送入模型。
4. 提供更细的无障碍设置：大字号、减少动画、高对比度。

## 不建议做的功能

- 不做“你是否抑郁”的二分类结论。
- 不做情绪聊天机器人替代专业支持。
- 不把语音概率转成医学诊断。
- 不根据单次录音生成强烈警示语。
- 不使用广告、埋点、第三方用户分析 SDK。
